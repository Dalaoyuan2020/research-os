## 为什么做这个

我是 **吕志远**,河海大学在读。做这个站,起因是一件让我有点困惑的小事。

我把 Claude Code 推荐给身边一些人,也手把手教过。但用了一阵子,他们好像没什么特别的感觉,人也没太大变化。可我自己用下来,却觉得几乎每周都在变强。

同样一个工具,差别怎么会这么大?

我想了很久。也许是我观察得不够,他们其实也在慢慢变;但我更相信,是我们用它的方式不一样——

大多数人把 AI 当一个「解题器」:有问题就问,拿到答案就关掉,题是解了,人还是原来的人。而我每次用它都会多走一步:读懂答案背后是怎么想出来的,把最值钱的那点东西抽出来,变成自己下次能用的本事。一道题做完,我比做之前强了一点点——日积月累,差距就拉开了。

后来我给这件事起了个名字:**「资源语言编程」**。GitHub 是全世界最好的开源工程库;在 Agent 时代,*能读懂、能把别人的好工程吸收成自己的能力,本身就是一种编程*。

Research OS,就是把这套「**发现 → 吸收 → 内化 → 自我增强**」的过程公开地做出来,也想找一些同频的人,一起把它做大。

## ① 基础 · 学术 AI 工具热榜

> 每天扫一遍 GitHub,把真正值得用的学术 AI 工具挑出来,按研究阶段分好,做成可读、可筛、可跳转的热榜。这是大家的共同起点。

- 🌐 **在线热榜**:https://dalaoyuan2020.github.io/research-os/
- 目前收录 **24+** 个项目、合计 **110k+** stars,按 `想法 / 实验 / 写作 / 综合` 四阶段分类
- 首页有一个可拖动的 **Skill 星球**:节点大小=热度、颜色=阶段、点击直达对应仓库
- 收录的不是"又一个 awesome 列表",而是**带吸收角度**的精选 —— 每个工具标注它最值得学的那一点

热榜由发现引擎持续更新,详见 [`docs/how-the-board-works.md`](./docs/how-the-board-works.md)。

## ② 框架 · Research OS 五层

在"用好这些工具"之上,我把自己的科研方法蒸馏成一套**五层系统**,把零散工具焊成一条流水线:

| 层 | 角色 | 做什么 |
|---|---|---|
| **L1 文献阅读** | 读 | 文献检索、结构解析、速读卡 |
| **L2 知识库** | 沉淀 | 跨文献沉淀知识网络(Karpathy Wiki 式双链) |
| **L3 文献写作** | 写 | 改稿版本管理、模拟期刊评审、引用规范、去 AI 味、配图 |
| **L4 自动科研** | 验证 | 实验驾驭(Harness)、自迭代循环 |
| **L5 项目管理** | 管理 | 多项目看板、进度总控、任务编排 |

一句话闭环:**读 → 沉淀 → 写 → 验证 → 管理**。系统本身也在持续吸收热榜里别人的好工程,自我迭代。

> 五层的展开、每层用到的真实 Skill、以及它和"资源语言编程"的关系,见 [`docs/five-layers.md`](./docs/five-layers.md)。

## ③ 案例 · 从一个想法到一篇论文

> 用各种 Skill 收集想法 → 编排这些 Skill → 达到预想的结果。

- **一个想法**:「某个预测任务,能不能更准、更说服人?」
- **收集**:文献检索 + 文献分析,对标目标期刊的真实惯例
- **编排**:改稿版本管理 + 模拟期刊评审 + 去 AI 味,串成一条改稿闭环
- **结果**:用数据(不是感觉)选出最稳的方法,改成一篇能投出去的论文

完整复盘见 [`docs/cases/from-idea-to-paper.md`](./docs/cases/from-idea-to-paper.md)。(更多案例持续补充。)

## 🧭 怎么用

1. **逛热榜** → 打开[在线站点](https://dalaoyuan2020.github.io/research-os/),按你当前的研究阶段筛,点进感兴趣的仓库。
2. **不止于用** → 别只把工具当"解题器"。挑一个,读懂它的工程,抽出内核,变成你自己的能力 —— 这才是 Research OS 的用法。
3. **搭你自己的** → 参考[五层框架](./docs/five-layers.md),把你常用的工具焊成一条属于你的流水线。

## 🤝 一起共创

这不是一个人的收藏夹,是想和同频的人一起长的东西。

- **发现了好用的学术 AI 工具?** 开一个 [Issue](https://github.com/Dalaoyuan2020/research-os/issues/new) 告诉我 —— 附仓库链接 + 一句话价值 + 它属于哪个阶段。
- **懒得手写?** 直接让你的 **AI Agent 替你提 Issue**(我们对 Agent 友好,见 [CONTRIBUTING.md](./CONTRIBUTING.md))。
- **想深度共创**(改框架 / 写案例 / 加阶段)?也在 [CONTRIBUTING.md](./CONTRIBUTING.md) 里。

入选的工具会进入热榜,并署名来源。
