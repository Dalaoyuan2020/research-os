# 理念 · 知识飞轮

> Research OS 的灵魂。如果只读一篇,读这篇。

## 一个让我困惑的观察

我把 Claude Code 推荐给身边一些人,也手把手教过他们。

但用了一阵子,我发现他们好像没什么特别的感觉,人也没太大变化——工具是会用了,可整个人看不出明显不同。

而我自己用下来,却觉得几乎每周都在变强,像被它推着往前走。

同样一个工具,差别怎么会这么大?

我想了很久。也许是我观察得不够,他们其实也在慢慢变;但我更相信,是我们用它的方式不一样。

大多数人把 AI 当一个「解题器」:遇到问题就问,拿到答案就关掉。题是解了,人还是原来的人,那些答案背后的思路,随着对话框关闭一起消失了。

而我每次用它,都会多走一步:读懂它给的答案是怎么想出来的,把最值钱的那点东西抽出来,变成自己下次能用的本事。一道题做完,我比做之前强了一点点。

这一点点,日积月累,就是天壤之别。

## 工具只是工具

会用一个工具,提升的是"这一次的效率"。
工具会过时,API 会变,模型会换代。三个月后,今天最火的 Skill 可能就没人提了。

那什么不会过时?

> **把别人最好的工程,快速读懂、抽出内核、变成自己的能力,然后不断迭代。**

这件事,我把它叫 **知识的自我迭代** —— 它像一个越转越快的**知识飞轮**(也是这个项目 logo 的意义)。

## 什么是知识飞轮

GitHub 是全世界最好的开源工程资源库。每天都有人把最聪明的做法、最干净的架构、最实战的踩坑,公开放上去。

在 Agent 时代,你不需要从零写每一行代码。你需要的是一种能力:

- **发现** —— 在海量仓库里,认出真正值得学的那几个;
- **吸收** —— 读懂它的工程,不是抄,而是看懂它为什么这么设计;
- **内化** —— 抽出可迁移的内核,变成你自己的方法;
- **自我增强** —— 把它接进你已有的系统,让系统比昨天更强一点。

*这四步连成一个飞轮:每转一圈,你的能力就强一点,下一圈就转得更快。* 你吸收的不是某一个工具,而是"全世界最好的工程"。

## Research OS 就是把这件事做出来

- **① 热榜**(基础)= "发现"这一步的公共基础设施 —— 替所有人收口,每天挑出值得吸收的。
- **② 五层框架**(我的实践)= "内化 + 自我增强"的载体 —— 一个真实在长的科研系统,持续把热榜里的好东西接进来。
- **③ 案例** = 证明它真的跑得通 —— 从一个想法,到一篇投出去的论文。

热榜本身,也用同一套逻辑自我迭代:发现引擎扫 GitHub → 新工具入库 → 成为下一轮的种子。**系统在公开地自我增强。**

## 一句话

> 工具会过时,「把别人最好的工程变成自己的能力」不会。

这就是为什么有这个仓库。也是为什么,它想找一些同频的人,一起长。
