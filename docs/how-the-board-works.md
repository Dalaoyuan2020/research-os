# 热榜怎么收录与更新

> 学术 AI 工具热榜不是一份静态的 awesome 列表,而是一个**会自循环的发现系统**。

## 三步循环

```
   扫 GitHub          人/Agent 复核           入库 → 上线
  发现引擎  ──────→   真实可用?学术相关?  ──────→  进热榜,成为下一轮种子
 (关键词搜索)         非营销?有吸收价值?         (署名来源)
```

1. **发现** —— `peers/discover.sh` 按多组关键词扫 GitHub,新仓库落 `peers/seen.txt` 去重。
2. **复核** —— 人工或 Agent 核一遍:真实可用、和科研相关、不是纯营销、有值得学的内核。
3. **入库** —— 合适的进 `data/repos.json`(单一真源),重建站点上线。**入选的工具会成为下一轮发现的种子。**

## 收录标准

| 要 | 不要 |
|---|---|
| 真实可用、和学术科研相关 | star 高但与学术无关 |
| 有明确的**吸收角度**(最值得学的那一点) | 纯转载 / 纯营销 / 无说明的堆叠 |
| 一个工具讲清一件事 | 一次塞一长串列表 |

## 每个工具记什么

不只是 star 数。每条记录带:

- `stages` —— 属于哪个研究阶段(想法 / 实验 / 写作 / 综合)
- `typ` —— 类型(Skill 库 / 写作 Skill / 自动科研 …)
- `one` —— **一句话:它最值得学 / 最能帮上忙的那一点**(这是核心,不是简单复制仓库简介)
