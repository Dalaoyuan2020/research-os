# 共创指南 · Contributing

> 写给**共创者**。Research OS 不是一个人的收藏夹,是想和同频的人一起长的东西。
> 你可以贡献「一个工具」,也可以贡献「一段框架 / 一个案例」。两条路都欢迎。

---

## 路线一:推荐一个学术 AI 工具(最轻量)

发现了好用的学术 AI 工具 / Skill / 仓库?**开一个 Issue 就够了。**

1. 进 [New Issue](https://github.com/Dalaoyuan2020/research-os/issues/new),选「推荐工具」。
2. 给齐三样东西:
   - **仓库链接**(GitHub 优先)
   - **一句话价值**:它最值得学 / 最能帮上忙的那一点
   - **研究阶段**:`想法` / `实验` / `写作` / `综合` 之一

我会核一遍(真实可用、和学术科研相关、不是纯营销),合适就进热榜,并**署名来源**。

### 让你的 AI Agent 替你提 Issue 🤖

我们对 Agent 友好。你可以直接让 Claude / 你的 Agent 替你提,把下面这段丢给它:

```
帮我去 github.com/Dalaoyuan2020/research-os 提一个 Issue,推荐一个学术 AI 工具。
工具仓库: <链接>
一句话价值: <它最值得学的一点>
研究阶段: <想法/实验/写作/综合>
标题写成 "[推荐] <工具名>",正文按上面三项列清楚。
```

> 之所以支持这种方式,正是因为这个项目的理念 —— **知识飞轮**:让 Agent 去发现、去吸收、去交流,本身就是在帮你把飞轮转起来。

---

## 路线二:深度共创(框架 / 案例 / 内容)

想改框架、写案例、补一个研究阶段?走 Pull Request。

- **加 / 改一个案例** → 在 `docs/cases/` 下新建一篇,讲清「想法 → 收集 → 编排 → 结果」。
- **完善五层框架** → 改 `docs/five-layers.md`,补你这层用到的真实 Skill 和踩坑。
- **改进热榜的吸收角度** → 不只是加 star 数,而是补"这个工具最值得学的内核"。

PR 规范:一个 PR 只做一件事;标题写清意图;正文说清「改了什么、为什么」。

---

## 我们看重什么 / 不看重什么

| 看重 | 不看重 |
|---|---|
| 真实可用、和科研相关 | star 数高但和学术无关 |
| **吸收角度**:它最值得学的内核是什么 | 纯转载、纯营销、无说明的堆叠 |
| 一个工具讲清一件事 | 一次塞一长串"awesome 列表" |

## 行为准则

就一条:**真诚、就事论事、尊重彼此的时间。** 我们在公开地学习别人最好的工程,也请把这份认真带进每一次交流。

---

有任何拿不准的,先开 Issue 聊。谢谢你愿意一起把它做大。 🌱
